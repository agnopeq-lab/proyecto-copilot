1. Introducción a los Grafos: Nodos, Aristas y Tipos

Un grafo es una estructura de datos compuesta por un conjunto de nodos (también llamados vértices) y un conjunto de aristas (o ejes) que conectan pares de nodos. Los grafos se utilizan para modelar relaciones y conexiones en redes sociales, mapas, rutas, y mucho más.

Aplicaciones de los grafos:

  • Redes sociales (usuarios y sus conexiones).
  • Mapas de carreteras (ciudades y caminos).
  • Redes eléctricas y de agua.
  • Biología (redes neuronales, relaciones entre especies).
  • Internet (enlaces entre páginas web).
  • Nodos (Vértices): Elementos o puntos del grafo.
  • Aristas (Ejes): Conexiones entre los nodos.

Tipos de grafos:

  • Grafo dirigido: Las aristas tienen dirección.
    Ejemplo: Twitter, donde A sigue a B, pero B no necesariamente sigue a A.
  • Grafo no dirigido: Las aristas no tienen dirección.
    Ejemplo: Facebook, donde la amistad es mutua.
  • Grafo ponderado: Las aristas tienen un peso o valor asociado.
    Ejemplo: Mapa de carreteras, donde el peso es la distancia entre ciudades.

2. Representación de Grafos

Existen dos formas principales de representar un grafo en la memoria de una computadora:

Lista de Adyacencia

Utiliza una lista para cada nodo, donde se almacenan los nodos a los que está conectado. Es eficiente en espacio para grafos dispersos.

  • Ventajas: Ocupa menos memoria en grafos con pocas conexiones (dispersos).
  • Desventajas: Consultar si existe una arista entre dos nodos puede ser más lento.
// Grafo no dirigido
A: B, E
B: A, C, E
C: B, E
D: E
E: A, B, C, D

Matriz de Adyacencia

Utiliza una matriz donde las filas y columnas representan los nodos. Un valor 1 indica la presencia de una arista. Es eficiente para consultar aristas.

  • Ventajas: Permite saber rápidamente si existe una arista entre dos nodos.
  • Desventajas: Ocupa más memoria, especialmente en grafos grandes y dispersos.
    A B C D E
A [0 1 0 0 1]
B [1 0 1 0 1]
C [0 1 0 0 1]
D [0 0 0 0 1]
E [1 1 1 1 0]

3. Algoritmos Fundamentales de Recorrido

Los algoritmos de recorrido permiten explorar todos los nodos y aristas de un grafo. Los dos más importantes son:

Búsqueda en Amplitud (BFS)

Explora el grafo nivel por nivel, utilizando una cola. Es útil para encontrar la ruta más corta en grafos no ponderados.

  • Aplicaciones: Encontrar la ruta más corta en mapas, redes sociales, laberintos.
  • Detectar si un grafo es conexo.
// Pseudocódigo BFS
BFS(grafo, nodo_inicial):
  crear una cola Q
  marcar nodo_inicial como visitado y ponerlo en Q
  mientras Q no esté vacía:
    nodo = Q.desencolar()
    para cada vecino de nodo:
      si vecino no ha sido visitado:
        marcar como visitado y poner en Q

Búsqueda en Profundidad (DFS)

Explora el grafo yendo lo más profundo posible antes de retroceder, utilizando una pila (o recursión).

  • Aplicaciones: Detectar ciclos en grafos, encontrar componentes conexos, resolver rompecabezas y laberintos.
  • Recorrer árboles de decisión o estructuras jerárquicas.
// Pseudocódigo DFS
DFS(grafo, nodo_inicial):
  marcar nodo_inicial como visitado
  para cada vecino de nodo_inicial:
    si vecino no ha sido visitado:
      DFS(grafo, vecino)
