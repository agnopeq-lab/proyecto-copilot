Post #2 — Representación de Grafos

Las dos formas principales de representar grafos en memoria son lista de adyacencia y matriz de adyacencia. Cada una tiene ventajas dependiendo de la densidad del grafo y las operaciones que se requieren.

Lista de adyacencia

Para cada nodo, se mantiene una lista de nodos adyacentes. Es eficiente en espacio para grafos dispersos (pocas aristas).

Ejemplo (para el mismo grafo del Post #1):

A: B, D
B: A, C, E
C: B, E
D: A, E, B
E: B, C, D

Matriz de adyacencia

Una matriz NxN donde la celda (i,j) indica si existe arista entre i y j (y opcionalmente el peso). Es más rápida para consultar si (u,v) existe, pero usa O(N^2) memoria.

Matriz para nodos [A,B,C,D,E] (0/1 para no ponderada):

ABCDE
A01010
B10101
C01001
D10001
E01110

Comparación rápida

  • Lista de adyacencia: buena para grafos dispersos — memoria O(V + E).
  • Matriz de adyacencia: buena para grafos densos o consultas rápidas de existencia de arista — memoria O(V^2).
